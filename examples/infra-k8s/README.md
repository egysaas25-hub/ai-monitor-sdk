# Infrastructure Example: Kubernetes

This folder provides a boilerplate example for deploying an AI Monitor instrumented microservice to Kubernetes.

## K8s Metadata Enricher

The `@momen124/ai-monitor-instrumentation` relies on Downward API environment variables to automatically identify the K8s pod and correlate incidents to namespace, node, and pod instances.

In the `deployment.yaml`, notice the usage of `fieldRef`:

```yaml
env:
  - name: POD_NAME
    valueFrom:
      fieldRef:
        fieldPath: metadata.name
  - name: POD_NAMESPACE
    valueFrom:
      fieldRef:
        fieldPath: metadata.namespace
```

If you deploy this, any alert generated by `AIMonitor` will automatically inherit `k8s.pod.name` and `k8s.namespace` metadata, vastly improving observability without manual tagging.

## Deployment

```sh
kubectl apply -f deployment.yaml
```
