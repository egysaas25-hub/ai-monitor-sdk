# PR Review Workflow

Use this workflow to review a pull request or staged change set.

1. Summarize the intended purpose of the change.
2. Check scope discipline: is the diff focused or mixing concerns?
3. Check correctness and root-cause handling.
4. Check tests, verification, and failure-mode coverage.
5. Check security, secrets handling, auth, permissions, and data ownership paths.
6. Check API and schema compatibility, migration readiness, and docs alignment.
7. Check technical debt handling: is any shortcut documented and tracked?
8. Check changelog quality and release impact.
9. Separate feedback into blockers, important follow-ups, and optional improvements.
10. End with a clear recommendation: approve, request changes, or approve with follow-ups.
